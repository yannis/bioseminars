The http://bioseminars.unige.ch/seminars url can take a variety of parameters to determine wich seminars to get from the database

  • Extension (.xml, .rss, .ics, nothing for html)
    • Define the format of output
    • Ex: http://bioseminars.unige.ch/seminars.xml => All seminars in xml formt
  • categories=
    • determine the categories which seminars have to be shown
    • it should be followed by the categories id number separated by “+”
    • Ex: http://bioseminars.unige.ch/seminars.xml?categories=1+2
  • scope=
    • could take “all”, “past” or “future” as value
    • will respectively get “all”, “past” or “future” seminars
    • Ex: http://bioseminars.unige.ch/seminars.xml?scope=past
  • before=
    • takes a date in the yyyymmdd format as value
    • will get only seminars started before or on this date
    • Ex: http://bioseminars.unige.ch/seminars.xml?before=20091231
  • after=
    • takes a date in the yyyymmdd format as value
    • will get only seminars started after or on this date
    • Ex: http://bioseminars.unige.ch/seminars.xml?after=20091231
  • internal=
    • will also show internal seminars if value is “true”
    • Ex: http://bioseminars.unige.ch/seminars.xml?internal=true
  • order=
    • takes “asc” or “desc” as value
    • respectively sort the list of seminars in ascendant or descendant order (for the start time value)
    • Ex: http://bioseminars.unige.ch/seminars.xml?order=asc
  • limit=
    • takes an integer as value
    • limit the number of returned seminars to the value of the limit parameter
    • Ex: http://bioseminars.unige.ch/seminars.xml?limit=20

All these parameters can be combined to get the desired seminars list.

  • Ex: http://bioseminars.unige.ch/seminars.xml?order=asc&scope=past&after=20090101&categories=1+2
